关于mac 文本转码问题

2018-05-25

简单的一个命令

实际场景：

需要手动把exccl表格，csv文件导入到mysql数据库，导入进去后发现出现乱码问题？

使用的平台：mac

【如果是window操作系统可以直接在「文件编辑器」中直接修改过来 ´д` ; 】

使用的工具：mac终端，可以使用默认自带的终端，我这里使用的是iterm

操作过程：

1.cd 到要转化的文件的位置

我的文件放在桌面，就可以如下：

cd Desktop

2.使用以下命令

iconv -c -f <源文件编码> -t <目标编码> <目标文件> > <转换后的编码文件>

我的是下面的

iconv -c -f GB18030 -t UTF-8 user1.csv > user0.csv

tip: 可以在文本编辑器中看到目标文件的编码, 使用偏好设置：


然后重新打开文件，就可以验证了。
